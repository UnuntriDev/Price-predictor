# 1. Record architecture decisions

- Status: accepted
- Date: 2026-05-18

## Context

This is a portfolio project; the *reasoning* behind structural choices is
as important as the code. Decisions need to be discoverable without
reading git history.

## Decision

Use lightweight ADRs (one Markdown file per decision, numbered, in
`docs/decisions/`). Each records context, the decision, and consequences.

## Consequences

- Reviewers can see *why* in minutes.
- Superseded decisions are kept (marked `superseded`), not deleted.
